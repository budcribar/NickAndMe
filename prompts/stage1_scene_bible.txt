Act as an expert screenwriter and narrative architect. Your task is STAGE 1 ONLY:
adapt the provided book into a **provider-agnostic scene bible** (JSON).

You are NOT writing video clips, timestamps, visual_prompt strings for Grok/Veo,
or continuation flags. Those are STAGE 2.

================================================================
STAGE BOUNDARY (HARD)
================================================================
OUTPUT = scene bible only (`schema_version`: "stage1.v1").
Do NOT emit `veo_clips`, `timestamp`, `visual_prompt`, `negative_prompt`,
or `veo_continuation_source`.

If the operator later changes VIDEO_PROVIDER or max clip length, Stage 1 must
still be valid without edits.

================================================================
GOALS
================================================================
1. Cover the book’s essential story in roughly {{TOTAL_RUNTIME_MINUTES}} minutes
   (operator/book estimate — picture books often 5–15 min, shorts shorter than features;
   sum of `duration_target_seconds` ≈ TOTAL_RUNTIME_MINUTES × 60 ±10%).
2. Break the story into scenes that a director could schedule (location + time + purpose).
3. Inside each scene, list ordered **story_beats** — concrete events we must see/hear —
   with relative `time_weight`, not wall-clock clip lengths.
4. Lock cast identity: seeds with description, portrait placeholder, **voice_profile**,
   **voice_label**, and display-name policy when names are withheld.
5. Lock **places** the same way: `location_seed_tokens` + per-scene `location_ids[]`
   (Phase 1 may keep multi-place scenes; do not invent new plot to force splits).
6. Preserve VO ↔ visual fidelity at the beat level (if the book says ball through window,
   the beat says through window — not “over a wall”).

================================================================
PRIORITY IF CONFLICTS
================================================================
1. Valid JSON matching stage1_scene_bible.schema.json
2. Narrative fidelity to the book
3. Runtime band (±10% of target)
4. Closed character world (no unnamed speakers)
5. Every recurring place has a stable Loc_* id

================================================================
CHARACTER SEEDS (REQUIRED)
================================================================
`global_production_variables.character_seed_tokens` must include every speaking or
recurring on-screen person (and age variants for flashbacks).

Each seed REQUIRED fields:
- description — **filmable identity** (species/age/build/hair/face/wardrobe + **stable colors/markings**
  that must not drift: e.g. “small black-and-white dog — never brown/tan redesign”)
- reference_image_placeholder — **bare filename only** under assets/characters/
  (e.g. `buster_ref.png`, `character_mom_ref.png`). NEVER a full path like
  `assets/characters/buster_ref.png` (the pipeline prefixes that folder).
- voice_profile (1–3 sentences: age/sex/pitch/pace/accent/energy + “same voice every scene”)
- voice_label

Also set when relevant:
- age_band, variant_of for Young/Teen
- display_name_policy: ok_anytime | withheld_until_reveal | never_on_screen
- name_reveal_note / canonical_given_name (adapter notes only; never force early on-screen names)
- name_reveal_scene (integer scene_number where on-screen/spoken given name first becomes allowed;
  required when display_name_policy is withheld_until_reveal)

**BOOK / PDF LIKENESS (when AVAILABLE_BOOK_IMAGES is provided in the user message):**
- For every **on-screen** character, set `source_image_pages`: array of 1–3 **page numbers**
  from AVAILABLE_BOOK_IMAGES where that character is clearly visible (cover + illustration
  pages for picture books). Example: `"source_image_pages": [1, 2, 5]`
- Prefer pages that show the face/body clearly over text-only pages.
- Off-screen only (display_name_policy never_on_screen / pure narrator): omit source_image_pages
  or use []. **Narrator does not need a reference image** — voice_profile + voice_label only;
  do not invent book plates or likeness for a voice-only narrator.
- Do NOT invent page numbers that are not listed in AVAILABLE_BOOK_IMAGES.
- The pipeline will copy those pages into assets/characters/*_bookref_* and set
  design_reference_images after Stage 1 — you only need correct page numbers (on-screen cast).

Age variants need their **own** voice_profile (child ≠ adult baritone).

================================================================
LOCATION SEEDS & PINS (REQUIRED — Phase 1 uses lists)
================================================================
Places drift like faces if they are free-text only. Stage 1 owns stable location ids.

1. **Global — `global_production_variables.location_seed_tokens`**
   Every recurring place (and important one-offs you will return to) gets a seed:

   - Key: stable id `Loc_*` (e.g. Loc_Kirk_Apt, Loc_Moxie_Gym, Loc_Joes_Bar)
   - display_name: director-facing name (“Kirk Street apartment”)
   - description: short place identity (architecture, era, city)
   - visual_lock: 1–3 sentences of **filmable** constants Stage 2 must inject
     (walls, furniture, window look, signage rules — not shot list)
   - reference_image_placeholder: optional plate path (e.g. assets/locations/loc_kirk_apt_ref.png)

   Prefer reusing ids when the same apartment / bar / gym returns across days.

2. **Per scene — `location_ids[]` (REQUIRED, non-empty)**
   Ordered list of Loc_* ids this scene uses (story order).

   - Single place: `["Loc_Kirk_Apt"]`
   - Multi-place (Phase 1, not yet split): `["Loc_Kirk_Apt", "Loc_Moxie_Gym"]`
     when the setting spans hard location changes inside one scene_number
   - Optional `primary_location_id`: dominant place for plates/lighting default
     (defaults to location_ids[0] if omitted)

3. **`setting` string** stays a human slug (Day + place names). It must be
   consistent with location_ids but does not replace them.

4. **Beats (optional)** may set `location_id` when a beat is clearly in one place
   inside a multi-place scene. Stage 2 uses this for clip pins.

5. **Do not split scenes solely for location in Phase 1** unless the operator asks.
   Tag multi-place scenes with multiple location_ids; later phases may split.

Rooms inside the same seed (kitchen vs living room of Loc_Kirk_Apt) share **one** id.

================================================================
SPOILERS & REVEAL TIMING (REQUIRED — Stage 1 owns this)
================================================================
Stage 1 MUST identify spoilers. Stage 2 and the verifier only enforce what you declare here.

1. **Per character (seeds)**
   - If the book withholds a legal/first name, secret identity, death, relationship, or plot twist
     that must not appear early: set display_name_policy / notes and **when** reveal is allowed
     (name_reveal_scene and/or name_reveal_note with beat language).
   - Prefer Character_* tokens in all early dialogue and visual descriptions.
   - **Never** ban a spoiler by writing the secret string into prompts (that teaches the model
     the secret). Describe the constraint without the payload
     (e.g. “no personal name on clothing or badges” — not “do not show the name X”).

2. **Global register (recommended)**
   In global_production_variables, when useful, set `spoiler_register` as an array of:
   - id (stable: sp1, sp2, …)
   - kind: name | identity | plot | relationship | death | other
   - summary (spoiler-safe wording — what is withheld, not the secret text if avoidable)
   - allowed_from_scene (first scene_number where reveal is OK; null = never on screen)
   - allowed_from_beat_id (optional finer grain once that scene exists)
   - characters (tokens involved)

3. **Per scene — spoiler_constraints[] (REQUIRED when any withhold applies)**
   Every scene that occurs **before** a reveal must list active constraints, e.g.:
   - "No name tags / personal names on clothing for Character_P (withheld_until_reveal)"
   - "Do not depict twist Y; still unknown at this story day"
   Scenes **at/after** reveal may drop the constraint or replace with
   "Reveal allowed: given name may appear in dialogue/props if the beat requires it."

4. **Per beat (when the beat is the reveal or must stay clean)**
   - On the reveal beat: intent/visual_event may allow the name; mark clearly (e.g. intent
     includes "name reveal").
   - On pre-reveal beats: must-not / visual_event must not depend on the secret.

If you cannot place a reveal scene from the book, set name_reveal_note to "unknown — keep
withheld" and treat all scenes as pre-reveal until the operator fills it.

================================================================
SCENE RULES
================================================================
For each scene:
- scene_number (1-based contiguous)
- scene_filename (filesystem-safe)
- setting, story_day, location_type
- **location_ids[]** (required Loc_* list — see LOCATION SEEDS)
- optional primary_location_id
- duration_target_seconds: integer 8–134 (story budget; Stage 2 will snap)
- dramatic_function, summary
- **characters_on_screen[] (REQUIRED, non-empty when anyone is visible)** —
  full list of **Character_*** tokens that may appear in **any** beat of this scene
  (hero + supporting). Picture books: include the animal/hero even when the beat is
  “Mom talks” if the hero is still on screen or reacts. Stage 2 + engine use this list
  to multi-lock reference images (not only the primary speaker).
- transition_type, lighting_continuity_token
- wardrobe_notes (incremental)
- spoiler_constraints[] (name tags, plot reveals, etc. — see SPOILERS section)
- music_intent (style + mood_arc; not a full timed song_structure unless easy)
- story_beats[] ordered
- source_book_refs[] and, when the operator supplies book text/PDF extracts,
  source_excerpts[] with short supporting passages (file/page + quote). Prefer
  quotes that justify visual_event and dialogue; do not invent book text.

Do NOT put camera-technical prompt endings like " / 720p, 24fps" in Stage 1.

================================================================
STORY BEAT RULES (CRITICAL)
================================================================
Each beat is one clear narrative/visual unit Stage 2 may map to one or more clips.

Required on every beat:
- beat_id (stable: b1, b2, …)
- intent
- visual_event (concrete SEE language)
- shot_scale_hint (soft)
- action_class: hold | dialogue | small_motion | big_action | establishing |
  hard_cut | flashback_enter | flashback_exit | montage
- continuity: new_setup | continuous_from_previous_beat | return_to_present | parallel
- time_weight (default 1; raise for important action/dialogue)

Audio on beat (REQUIRED shape — Stage 2 maps these into clip `audio_payload`):
Put these **on the beat root** (preferred). Nested `audio: { ... }` is also accepted, but
do **not** put dialogue only inside free-text intent.

- delivery: spoken_on_camera | voiceover_internal | none
- speaker: Character_* or "none"
- dialogue: text (may be long; Stage 2 may split across clips without inventing new plot)
- ambient_or_sfx optional (short Foley / room tone intent)

**Picture-book / narrator VO (critical for automation):**
- Off-screen book narrator: speaker=`Character_Narrator` (or your narrator seed),
  delivery=`voiceover_internal`, dialogue = the spoken line (clean book text).
- On-screen characters still get tokens in `characters_on_screen` and beat `primary_subject`
  even when the narrator is speaking over them.
- Never set delivery=`none` when the book has a line for this beat.

**Multi-character dialogue (critical for automation):**
- **Bouncing conversation:** prefer **one beat** with  
  `dialogue_turns: [{ "speaker": "Character_Mom", "dialogue": "…", "delivery": "spoken_on_camera" }, …]`  
  so Stage 2 can keep banter on one continuous clip. Root `speaker`/`dialogue` may be the
  first turn or a short summary; turns are the source of truth.
- Split into **separate beats** when the story needs a hard cut, place change, or long gap —
  not merely because speakers alternate.
- When anyone is `spoken_on_camera` with 2+ characters on screen, set primary_subject to the
  main on-screen focus (often the first speaker), list listeners in characters_on_screen,
  and note: only the active speaker’s mouth moves each turn; pets/dogs keep snout closed.

Blocking:
- If someone confronts / approaches / answers at a window or doorway, put eyeline notes in
  blocking_notes (who faces whom, camera side). Do not leave “steps toward her” alone.

Must-not:
- List forbidden visuals that would break VO or spoilers (name badge, wrong prop outcome,
  early depiction of a registered spoiler). Prefer spoiler-safe phrasing (no secret payload).
- For animal/hero identity: ban recolors that break the seed (e.g. “not a brown dog if seed is black-and-white”).

primary_subject (REQUIRED whenever anyone is on screen):
- **On-screen visual hero** Character_* token Stage 2 puts **first** in visual_prompt for
  identity lock — even when speaker is the narrator VO.
- Do **not** set primary_subject to the off-screen narrator.

Also list **other on-screen** characters in visual_event using Character_* tokens when they
share the frame (e.g. Mom and Buster both visible). Prose-only “the dog” is weaker for automation.

location_id (optional on beat):
- When the scene has multiple location_ids, set the beat’s location_id to the Loc_*
  where this beat happens so Stage 2 can pin each clip.

================================================================
RUNTIME
================================================================
Sum of duration_target_seconds ≈ total_runtime_target_seconds (±10%).
Prefer fewer longer scenes over dozens of 8s stubs unless the book needs mosaic.

If output would exceed token limits, deliver complete scenes from scene 1 and set:
- next_scene_number
- cumulative_duration_target_seconds

================================================================
SCHEMA HARD RULES (FAIL IF VIOLATED — post-processor may fix, but do it right)
================================================================
These match stage1_scene_bible.schema.json. Wrong types cause pipeline failure.

global_production_variables MUST always include:
- target_aspect_ratio (string, e.g. "16:9")
- resolution (string, e.g. "720p")
- frame_rate (integer 24 — NOT the string "24fps")
- directorial_treatment (string)
- total_runtime_target_seconds (integer = TOTAL_RUNTIME_MINUTES × 60)
- character_seed_tokens (object)
- location_seed_tokens (object)

Per scene:
- story_day: STRING only (e.g. "Day 1", "Night 3", "Flashback summer ~age 12").
  NEVER a bare number like 1 or 0.
- location_type: ONLY one of: int | ext | mixed | flashback | dream | montage
  NEVER "exterior", "interior", "interior_exterior_mix", "montage_mix", etc.
- music_intent: object with REQUIRED style_description (string). Optional mood_arc, vocal_style.
- location_ids: non-empty array of Loc_* strings when places are known.
- source_excerpts (if present): array of objects ONLY:
  { "source": "book page N or file name", "excerpt": "quote text" }
  NEVER bare strings. Prefer omit source_excerpts if unsure.

Character seeds:
- Required strings: description, reference_image_placeholder, voice_profile, voice_label
- reference_image_placeholder = bare filename only (see CHARACTER SEEDS)
- Optional fields: omit the key if unknown — do NOT emit null
  (no "canonical_given_name": null, no "name_reveal_scene": null)

Per scene cast:
- characters_on_screen: non-empty array of Character_* when the scene has on-screen beings
- Every beat with picture: primary_subject set; every beat with book speech: delivery/speaker/dialogue set

Location seeds:
- Required: display_name, description
- visual_lock: string recommended (filmable constants)

Do NOT invent Stage 2 fields anywhere: no veo_clips, visual_prompt, timestamp,
negative_prompt, veo_continuation_source on scenes or beats.

OUTPUT

Return ONLY valid JSON for schema_version "stage1.v1".
No markdown fences unless the host requires them.
No trailing commentary after the closing brace.

Operator supplies: book text, TOTAL_RUNTIME_MINUTES (from book-length analysis unless
overridden), any spoiler/name policies, and optionally existing character seeds to
extend rather than reinvent.

Also respect prompts/shared_rules.txt when present.
