# Shared rules (prompt pack)

Rules that Stage 1, Stage 2, and the clip verifier must all respect.
Prefer short, testable bullets. Do not dump one-off clip notes here — promote
only durable patterns (eyeline, continuous action, VO↔visual fidelity, age tokens).

When a rule is scene-specific, keep it out of this file and fix Stage 1/2 for that scene.

================================================================
BASELINE (always on)
================================================================
- **Spoilers:** Stage 1 owns reveal timing (`display_name_policy`, `name_reveal_scene`,
  per-scene `spoiler_constraints`, optional `spoiler_register`). Stage 2 must not put
  withheld given names or secret plot facts on screen/in dialogue before allowed.
  Verifier fails early name tags, spoken withheld names, or registered plot leaks.
  Never “ban” a secret by writing the secret string into prompts.
- **Tokens:** Use Character_* in plans; age variants for flashbacks.
- **VO↔visual:** If the beat/VO says through the window, the picture must not do over the wall.
- **Cast automation:** Stage 1 scenes list full `characters_on_screen`; Stage 2 copies it and
  names every still-visible Character_* in visual_prompt so the engine can multi-lock refs.
- **Audio automation:** Stage 1 beat root (or nested audio) carries delivery/speaker/dialogue;
  Stage 2 must map that into clip audio_payload — never drop book lines.
  Stage 2 must **name who talks and what they say**. Single speaker: root fields +
  saying: "…". Multi-speaker banter: one continuous clip with `dialogue_turns`
  (ordered speaker+line list) so conversation can bounce without a cut per line;
  only the active speaker lip-syncs each turn. Split clips only for hard cuts /
  new setups / length — not merely alternating speakers.
- **Identity stability:** character seed descriptions include stable colors/markings;
  Stage 2 restates them; engine locks reference plates — no free redesign between scenes.
- **Ref placeholders:** bare filenames only (`buster_ref.png`), not full assets/ paths.
- **Single speaker lip-sync:** on `spoken_on_camera`, only `audio_payload.speaker` mouths
  the line; dogs/pets and non-speakers stay mouth/snout closed (listening). No dual lip-sync.
- **Native Grok audio:** film dialogue/narration comes only from the video model.
  No post-process TTS / edge-tts / SAPI path.
