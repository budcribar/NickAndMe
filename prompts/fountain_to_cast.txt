Act as a casting director and continuity supervisor for a film adaptation.

Your task: read the FOUNTAIN screenplay (and optional BOOK text) and produce a
CLOSED CAST list as JSON only — for production pinning (portraits, book plates,
multi-ref video). Do NOT rewrite the screenplay.

================================================================
GOALS
================================================================
1. List every character who appears ON SCREEN or SPEAKS (including silent leads).
2. Include pure voice-only narrators when they have dialogue cues (NARRATOR, etc.).
3. Infer filmable looks from action description + book text (species, size, colors,
   wardrobe, age). Do not invent major plot or people who never appear.
4. Use stable keys: Character_<PascalOrSnake> (e.g. Character_Buster, Character_Momma).
5. Picture-book / silent heroes often only appear in action lines (e.g. "BUSTER, a
   small dog, runs…") — they MUST still get a seed even with no dialogue cue.
6. **BOOK-FIRST LOOKS:** When BOOK text is provided, mine it for age, build, hair,
   eyes, face, clothing, era, and distinguishing traits. description + visual_lock
   must be concrete enough to drive portrait generation and prevent role swaps
   (e.g. confessor Narrator ≠ elderly Old Man). Never leave "as described in the
   screenplay" or empty looks for on-screen roles when the book has cues.
7. **ON-CAMERA POV:** First-person confessors / presenters who face the camera are
   display_name_policy ok_anytime (NOT never_on_screen). never_on_screen only for
   pure off-screen VO with no body on camera.
8. **AUDIENCE / PERFORMANCE (from book, general — not a fixed eye recipe):** Infer how
   this adaptation relates to the viewer and how speaking characters typically engage
   (e.g. first-person confessional direct address, objective third-person observation,
   picture-book storybook staging, asides vs looking at scene partners). Put a short
   film-level lock in performance_lock. Per character, optional performance_notes when
   that role has a special address pattern. Do NOT hardcode "eyes open facing camera"
   for every project — only state conventions the book/screenplay actually support.

PRIORITY IF CONFLICTS:
1. Valid JSON matching the schema below
2. Fidelity to screenplay + book (on-screen cast complete; looks from book when present)
3. Closed cast (no unnamed "MAN #2")
4. One render medium for the whole film
5. Distinct faces for each on-screen Character_* (no shared generic pale man)

================================================================
OUTPUT (HARD) — JSON ONLY
================================================================
Return ONLY a JSON object (no markdown fences, no commentary):

{
  "schema_version": "cast_seeds.v1",
  "movie_title": "…",
  "render_style_lock": "STYLE LOCK: … one medium for all cast …",
  "performance_lock": "PERFORMANCE LOCK: … how this film addresses the viewer / default gaze & staging conventions inferred from book+screenplay …",
  "character_seed_tokens": {
    "Character_Example": {
      "description": "filmable identity: species/age/build/hair/face/wardrobe + stable colors",
      "visual_lock": "1–3 sentences of constants that must not drift between scenes",
      "wardrobe_always": ["optional free-text sticky items"],
      "canonical_given_name": "Display name",
      "display_name_policy": "ok_anytime | never_on_screen | withheld_until_reveal",
      "voice_label": "short label",
      "voice_profile": "1–3 sentences age/sex/pitch/pace/energy; same voice every scene",
      "performance_notes": "optional: how this role addresses camera or partners when speaking",
      "reference_image_placeholder": "character_example_ref.png",
      "source_image_pages": [2, 4]
    }
  }
}

FIELD RULES:
- description: 1–3 dense sentences of filmable identity (age band, build, face, hair,
  eyes, skin, wardrobe, era/medium). Enough for a portrait model with no other context.
- visual_lock: 1–3 sentences of **constants that must not drift** (same face, hair,
  signature garment/feature). Explicitly distinguish from other cast when similar.
- wardrobe_always: sticky garments/colors only (not full plot costume changes).
- description / visual_lock / wardrobe_always: **filmable only** — species, size, age,
  coat/hair colors, markings, face shape, real garments/props with color/shape.
- **FORBIDDEN in description/visual_lock:** "as described in the screenplay",
  "as in the book", "see screenplay", empty or name-only stubs for on-screen roles.
- **NICKNAMES vs VISUAL (HARD):** Book titles, epithets, and personality labels belong in
  dialogue or the movie title — **never** in description, visual_lock, or wardrobe.
  Bad: “noodle head look”, “noodle-head silhouette”, “biscuit-head hat”, “not very bright”
  as a facial design instruction (vague personality is not anatomy).
  Good: “small black-and-white dog, floppy ears, bright eyes, soft rounded head shape,
  short coat; snug black-and-white pajama top and bottoms when dressed.”
- Do **not** quote title words as if they were physical features.
- Metaphorical wardrobe: only real garments shown in art/story (e.g. actual pajamas), not
  coat-as-metaphor sleepwear inventing extra clothes the art never draws.
- display_name_policy never_on_screen: pure VO narrator — omit visual_lock or leave short.
- performance_lock (film-level): 1–3 sentences from BOOK + Fountain about audience relationship
  and default performance/staging (e.g. "first-person confessional often addresses implied
  listener/camera when speaking; other characters are observed objects"; or "objective
  picture-book staging; characters look at each other / scene action, not the viewer").
  Prefer conventions over anatomy recipes. Empty only if nothing can be inferred.
- performance_notes (per character, optional): only when that role differs from the film default.
- reference_image_placeholder: bare filename only under assets/characters/ (no folder path).
- source_image_pages: optional 1–3 page numbers from book --- PAGE N --- if book has plates
  and the character is clearly visible; else [].
- Include humans AND animals that appear on screen.
- Do NOT invent characters not supported by screenplay or book.
- Prefer names used in the Fountain (BUSTER, MOMMA, DADDY, NARRATOR).

================================================================
OUTPUT
================================================================
JSON only. No trailing commentary.
